# 广西新高考院校专业组志愿预测计划书原始归档

> 本文档用于完整记录用户提供的广西新高考院校专业组志愿预测方案。内容保留原计划的核心结构、政策口径、数据表设计、模型公式、执行流程、交付物、注意事项和虚构示例；涉及真实年份数据时，仅说明应如何取得、清洗、建模、回测，不编造真实投档数据。

广西从新高考后采用“院校专业组”作为志愿单位，本科普通批实行院校专业组平行志愿；官方解读明确，本科普通批、高职高专普通批设置 40 个院校专业组平行志愿，每个院校专业组内设置 20 个专业和 1 个是否服从专业组内调剂选项，且考生必须符合院校专业组的选科要求。广西平行志愿投档遵循“分数优先、遵循志愿”，按考生投档基准分、志愿顺序和院校专业组招生计划投档，普通类投档基准分为高考总分，即总成绩加政策性加分。

2024 年、2025 年的本科普通批院校专业组投档最低分已可从广西招生考试院发布及其转载页面核验，2024 年页面列出了“院校代码、院校名称、专业组、投档最低分、备注”等字段，2025 年页面也指向广西招生考试院完整表格。2024、2025 年一分一档表也应以广西招生考试院、阳光高考平台为准，2024 年阳光高考页面列出物理类一分一档链接，2025 年阳光高考也发布了广西普通高考普通类一分一档表物理类。高校招生章程应以教育部阳光高考平台和高校招生官网为主，教育部也明确招生章程主要查询途径包括阳光高考信息平台和高校官方招生网站。

截至 2026 年 5 月 9 日，不能把 2026 年本科普通批招生计划、一分一档、投档线当成已知数据；目前公开可检索到的是 2026 年普通高考报名、补报名、考试缴费等信息，正式预测必须等 2026 年招生计划和一分一档发布后再更新。

## 一、问题定义：这不是“分数线预测”，而是院校专业组风险决策建模

这个问题应定义为“面向广西物理类、物化生考生，在本科普通批院校专业组平行志愿规则下，对每个可报院校专业组的最低投档位次分布、录取概率、风险等级和可解释风险因素进行预测的排序决策问题”。

核心不是预测某个学校多少分，而是预测某个“院校专业组”在 2026 年广西物理类本科普通批中的最低投档位次区间，并判断一个考生的位次相对于该区间是冲、稳、保还是不建议。

应拆成五类任务。

### 第一，院校专业组最低录取位次预测

这是回归任务。目标变量是某院校专业组在广西物理类本科普通批的投档最低位次，记为：

```text
R_{g,y}
```

其中 `g` 表示院校专业组，`y` 表示年份。注意 `R` 是位次编号，数值越小，竞争越强。例如 8000 名优于 15000 名。模型输出不应是单点，而应是区间：

```text
[\hat R_{g,y}^{10%}, \hat R_{g,y}^{50%}, \hat R_{g,y}^{90%}]
```

### 第二，院校专业组最低位次变化方向预测

这是分类任务。目标是判断专业组最低位次相较上一年是前移、稳定还是后移：

```text
D_{g,y} =
前移,  if R_{g,y} < R_{g,y-1}(1-\tau)
稳定,  if |R_{g,y}-R_{g,y-1}| <= \tau R_{g,y-1}
后移,  if R_{g,y} > R_{g,y-1}(1+\tau)
```

其中 `\tau` 可取 3%、5% 或由回测误差分布确定，不能拍脑袋固定。

### 第三，考生对某专业组的录取概率预测

这是概率预测任务。设考生 2026 年位次为 `r_c`，专业组最低投档位次的随机变量为 `T_g`。考生能被该专业组投档的条件是：

```text
r_c <= T_g
```

因此录取或投档概率可定义为：

```text
P_g(admit) = P(T_g >= r_c)
```

这一步必须输出概率区间，例如“约 35%—50%”，不要输出“47.3%”这种伪精确数字。

### 第四，冲、稳、保、垫志愿分层

这是决策分类任务，不是模型本身。它以预测位次区间、概率、专业偏好、调剂风险、章程限制为输入，输出“冲、稳中有险、稳、保、垫底保底、不建议、数据不足”。

### 第五，专业组风险解释模型

这是解释任务。模型要回答：为什么某专业组风险上升？是因为计划减少、热门专业占比提高、组内新增计算机、从冷门组拆出热门组、城市热度上升、学校层级上升，还是中外合作/民族班/专项计划口径不同造成不可比？

为什么以位次为核心，而不是分数？因为高考分数受当年试卷难度、赋分分布、考生群体变化影响，跨年不可直接比较；而投档过程本质上是按投档基准分排序后依次检索院校专业组，位次更接近投档机制。广西官方政策也明确平行志愿按投档基准分排序、按志愿顺序检索投档，院校专业组是基本投档单位。分数只用于换算位次，不能作为跨年预测主变量。

## 二、需要收集的数据

数据分四层：必须数据、强烈建议数据、辅助数据、人工标注数据。对目标来说，必须严格限定广西、物理类、物化生可报、本科普通批、普通类院校专业组，不要把历史类、旧高考理科、提前批、专项计划、中外合作、民族班、预科、征集志愿混入同一模型。

### 必须数据

必须数据包括：

- 2024、2025、2026 年广西物理类一分一档表。
- 2024、2025 年本科普通批院校专业组投档最低分。
- 由一分一档换算出的 2024、2025 年最低投档位次。
- 2024、2025、2026 年招生计划。
- 院校代码、院校名称、专业组代码、专业组内专业、每个专业计划数、再选科目要求、批次、科类、计划性质。

来源应以广西招生考试院、广西阳光志愿信息服务系统、广西高考指南、阳光高考平台为主。获取方式可以是官方 Excel、PDF、网页表格、官方系统导出或人工录入。清洗时要统一年份、科类、批次、院校代码、专业组代码，处理空白投档、无出档、征集志愿、降分投档等特殊标记。

口径问题主要是：一分一档有“全国性加分”“地方性加分最高分”等版本，必须与投档线使用的投档基准分口径一致；投档线只有分数时，要通过同年同科类一分一档换算为位次；同分考生位次有区间，不能假装精确到唯一名次。

### 强烈建议数据

强烈建议数据包括：

- 985、211、双一流标识。
- 学校所在省份、城市、城市层级。
- 学费、校区、身体条件限制、单科成绩限制、外语语种限制。
- 高校招生章程特殊录取规则。
- 专业组拆分、合并、新增、更名记录。
- 专业组内专业类别和热门专业占比。

这些数据适合机器学习，也适合人工规则，因为它们能解释位次变化。章程限制必须从阳光高考平台或高校官网核验，不能只看第三方摘要。

### 辅助数据

辅助数据包括：

- 第三方网站整理的投档线、招生计划、专业热度。
- 就业趋势、薪酬报告、搜索指数。
- 学校排名、学科评估、软科/校友会等榜单。

这些数据只能辅助打标签，不能作为唯一依据。尤其是投档线和计划数，一旦第三方与官方冲突，必须以官方为准。

### 人工标注数据

人工标注数据包括：

- 专业热度标签。
- 冷门风险标签。
- 是否强双非。
- 是否明显强势专业。
- 是否专业禁忌。
- 是否适合保底。
- 是否调剂风险高。
- 是否需要单独建模。

人工标签要有规则，不可凭感觉。例如“计算机/软件工程/人工智能/电子信息/自动化/通信”标为热门工科；“护理、农林、矿业、地质、土木、材料、化工、环境、生物、食品、海洋”等按个人偏好标为谨慎或禁忌；“中外合作、民族班、专项计划、预科班”标为特殊口径，不进入普通组模型。

数据是否能用于 2026 预测，要分清两类。2024、2025 历史投档线和计划可用于训练、回测、建立先验；2026 招生计划和一分一档表只能在官方发布后用于最终预测；2026 投档线和录取结果是事后结果，只能用于回测，不能用于填报前建模。

## 四、基础公式与统计模型

先定义符号。`R_{g,y}` 是专业组 `g` 在年份 `y` 的最低投档位次，数值越小越热门；`P_{g,y}` 是招生计划数；`H_{g,y}` 是热门专业占比；`C_{g,y}` 是城市热度；`U_g` 是院校层级特征；`M_{g,y}` 是专业组结构变化特征；`X_{g,y}` 是全部特征向量。

### 1. 去年位次延续模型

```text
\hat R_{g,2026}^{base}=R_{g,2025}
```

适用场景：专业组结构稳定、招生计划变化不大、专业构成变化不大。优点是解释性强、难以过拟合。缺点是不能捕捉计划变化、专业组拆分、热门专业加入等冲击。误差来源包括 2025 年偶然波动、同分位次不精确、官方分数转位次误差。

权重估计方式：它本身无权重，但可在回测中作为基线。任何复杂模型都必须先证明比它更好。

### 2. 两年趋势模型

建议用对数位次建模：

```text
\Delta \log R_{g,2025} = \log R_{g,2025} - \log R_{g,2024}

\hat{\log R}_{g,2026} =
\log R_{g,2025} + \lambda \Delta \log R_{g,2025}

\hat R_{g,2026} = \exp(\hat{\log R}_{g,2026})
```

其中 `\lambda` 是趋势延续强度。不能直接取 1，应通过回测或贝叶斯收缩估计。由于广西新高考专业组口径只有 2024、2025 两年可比数据，2026 填报前对 `\lambda` 应保守，建议初始设为 0.2—0.5，并用 2026 录取结果后再校准。若专业组发生拆分、合并、新增，则两年趋势模型失效。

### 3. 招生计划修正模型

计划数增加通常会使最低位次后移，即 `R` 变大；计划数减少通常会使最低位次前移，即 `R` 变小。但影响幅度必须估计。

```text
\hat{\log R}_{g,y}
=
\alpha
+ \beta \log R_{g,y-1}
+ \gamma \log(P_{g,y}/P_{g,y-1})
+ \epsilon
```

若 `\gamma > 0`，说明计划增加会使最低位次数值变大。适用于同一专业组结构稳定、计划变化可比的情况。缺点是当专业组同时换专业、换校区、变学费时，计划效应会和其他因素混杂。

`\gamma` 不能拍脑袋，应在 2024 预测 2025 的回测中估计；如果样本不足，可按院校层级、专业组热度、计划规模分层估计。例如 211+热门组、普通本科组、中外合作组分别估计。

### 4. 专业热度修正模型

先定义热门专业占比：

```text
H_{g,y} =
\frac{\sum_{m \in g} plan_{m,y} \cdot hot_m}
{\sum_{m \in g} plan_{m,y}}
```

其中 `hot_m` 是专业热度标签，可取 0、0.5、1，或按“计算机、电子信息、人工智能、软件工程、通信、自动化”等设高值。

模型：

```text
\hat{\log R}_{g,y}
=
\hat{\log R}_{g,y}^{base}
+ \theta_H (H_{g,y}-H_{g,y-1})
+ \theta_{H0}H_{g,y}
```

如果热门专业占比提高，理论上最低位次更容易前移，`\theta_H` 通常应为负。但必须通过回测估计，不能预设。适用场景是专业组专业构成变化明显。缺点是“热度”标签有主观性，必须有标注规则和复核。

### 5. 院校层级修正模型

院校层级可编码为：

```text
Tier_g =
a_1 I(985)
+ a_2 I(211)
+ a_3 I(双一流)
+ a_4 I(强双非)
```

但更建议不用人工指定 `a`，而在回归或树模型中作为特征估计。可解释模型为：

```text
\hat{\log R}_{g,y}
=
...
+ \delta_1 I(985)
+ \delta_2 I(211)
+ \delta_3 I(双一流)
+ \delta_4 I(强双非)
```

层级修正适合捕捉“同等专业、同等城市下，学校品牌对位次的吸引力”。缺点是 985、211、双一流之间高度相关，线性模型容易多重共线。可用岭回归或层级贝叶斯缓解。

### 6. 城市热度修正模型

城市热度可用城市层级、区域、是否一线/新一线、省内外、离广西距离、就业吸引力等构造。模型：

```text
\hat{\log R}_{g,y}
=
...
+ \eta C_{city(g)}
```

若城市越热，最低位次通常越前移，因此 `\eta` 可能为负。全国范围内，北京、上海、广州、深圳、杭州、南京、苏州、成都、武汉、西安等城市的热度通常较高；东北、西北、部分非省会城市热度可能较低，但强校强专业会抵消地域冷度。该部分应以回测估计为准，不能简单按城市印象打分。

### 7. 专业组结构变化修正模型

专业组拆分、合并、新增是新高考模型最大风险之一。先定义专业集合相似度：

```text
Jaccard(g_y,g_{y-1}) =
|Majors_{g,y} ∩ Majors_{g,y-1}| /
|Majors_{g,y} ∪ Majors_{g,y-1}|
```

也可用按计划数加权的重合度：

```text
Overlap_w =
\sum_m min(share_{m,y}, share_{m,y-1})
```

规则如下：若 `Overlap_w >= 0.8`，视为结构稳定；0.5—0.8，视为轻度变化；0.2—0.5，视为重度变化；低于 0.2 或无历史对应组，视为新增/不可直接比较。

修正模型：

```text
\hat{\log R}_{g,y}
=
...
+ \phi_1 I(stable)
+ \phi_2 I(split)
+ \phi_3 I(merge)
+ \phi_4 I(new)
```

对于新增组，不应强行套去年位次，应使用“相似专业组匹配”：同校相似专业组、同层级同城市相似专业组、同专业类别全国在广西历史表现，生成宽区间。

### 8. 综合风险评分模型

最终不要用一个神秘的“压力指数”，而应从概率和不确定性出发。设模型给出最低位次分布 `T_g`，考生位次为 `r_c`：

```text
p_g = P(T_g >= r_c)
```

设区间宽度为：

```text
uncertainty_g = (Q90_g - Q10_g) / Q50_g
```

设政策/章程风险为 `policy_g`，取 0—1；专业禁忌惩罚为 `forbid_g`，取 0—1。

综合风险分：

```text
RiskScore_g
=
100(1-p_g)
+ w_u * uncertainty_g
+ w_p * policy_g
+ w_f * forbid_g
```

其中 `w_u,w_p,w_f` 必须通过回测和人工校准确定。若没有足够历史样本，先不把它作为主要排序依据，只作为解释辅助。最终志愿排序应同时看录取概率、专业接受度、调剂风险和院校层级。

## 五、机器学习模型设计

这个任务需要机器学习，但机器学习只能做辅助，不应替代规则和人工核验。原因是广西新高考院校专业组模式的可比历史年份很少，2024、2025 是关键数据，2026 填报前可用于时间回测的数据有限。机器学习适合发现计划变化、专业组结构、城市、院校层级、专业热度之间的非线性关系；不适合在缺乏口径一致历史数据时“自动算准某个组的最低位次”。

模型比较如下。

线性回归适合预测 `\log R_{g,y}` 或 `\Delta \log R_{g,y}`。优点是可解释，缺点是处理非线性能力弱。

岭回归和 Lasso 适合在特征较多、共线性较强时使用。岭回归能稳定 985、211、双一流、城市热度等高度相关特征；Lasso 可做特征筛选。但 Lasso 可能误删有解释价值的特征。

随机森林适合捕捉非线性和特征交互，例如“211+冷门城市+热门专业组+计划减少”。缺点是样本少时容易过拟合，输出概率和区间需要额外校准。

XGBoost 适合预测位次变化率、风险等级、方向分类。优点是效果强，缺点是参数敏感，需要严格回测和特征泄漏检查。

LightGBM 适合样本量较大、类别特征较多、训练效率要求高的场景。可用于预测 `\Delta \log R`、方向分类、风险等级。但若只有广西两年数据，必须限制树深、学习率和叶子数。

CatBoost 适合处理院校、城市、专业类别等类别特征，减少手工编码压力。对于“院校代码、城市、学校类型、专业标签”多的任务较友好。

分位数回归适合输出预测区间，例如 10%、50%、90% 分位位次。它比单点预测更适合志愿填报，因为考生需要的是风险区间。

贝叶斯回归适合小样本和不确定性表达。可把“去年位次延续”作为先验，再让计划变化、专业热度、城市热度做修正。优点是稳健，缺点是实现和解释成本稍高。

分类模型适合预测“前移、稳定、后移”。它不直接给最低位次，但能提示某专业组是否有突然上涨风险。

概率校准模型用于把模型输出转成可信概率。可以使用 Platt scaling、isotonic regression 或按分箱校准。没有校准的机器学习概率不能直接用于志愿表。

推荐组合是：

- 朴素基线模型作为底线。
- 岭回归或贝叶斯回归作为可解释主模型。
- LightGBM/CatBoost 作为非线性辅助模型。
- 分位数回归输出区间。
- 概率校准模型输出录取概率。
- 最后用人工规则处理中外合作、专项、民族班、预科、新增组、拆分组合并、章程限制和专业禁忌。

## 七、回测与验证

严谨回测至少分三层。

### 第一层：用 2024 年数据预测 2025 年

输入只能使用 2024 投档线、2024 一分一档、2024 招生计划、2025 招生计划、2025 专业组结构和 2025 填报前可知特征。目标是预测 2025 年最低投档位次。不能使用 2025 投档线、2025 征集、2025 录取结果作为特征。

### 第二层：2026 结果出来后的真正两年趋势回测

等 2026 结果出来后，用 2024、2025 数据预测 2026，做真正的两年趋势回测。这一步能检验两年趋势模型是否可靠。

### 第三层：分层回测

至少分为：

- 211 及以上院校专业组。
- 热门工科专业组。
- 计划大幅变化组。
- 新增/拆分/合并组。
- 广西区内组。
- 省外组。
- 中外合作组。
- 冷门保底组。

评价指标如下。

最低位次误差：

```text
MAE = (1/n) * \sum_g |\hat R_g - R_g|

RMSE = sqrt((1/n) * \sum_g(\hat R_g - R_g)^2)

MAPE = (1/n) * \sum_g |\hat R_g - R_g| / R_g
```

区间覆盖率：

```text
Coverage = (1/n) * \sum_g I(Q10_g <= R_g <= Q90_g)
```

若输出 80% 预测区间，覆盖率应接近 80%。如果只有 50% 覆盖，说明区间过窄、模型过度自信。

冲稳保分类错误率：

```text
ErrorRate =
风险档判断错误的专业组数 / 总专业组数
```

对个人志愿更重要的是“保底失效率”：

```text
FailSafeRate =
被模型判为保或垫底但实际未达到投档线的组数 /
被模型判为保或垫底的组数
```

以及“冲刺命中率”：

```text
HitRushRate =
被判为冲且实际达到投档线的组数 /
被判为冲的组数
```

重大误判案例必须单独复盘。比如某 211 热门工科组计划减少 30%、专业组从大类拆出计算机、城市热度高、学费变化、校区变化、章程限制变化、某强校新开人工智能专业，这些都可能造成暴涨。

什么结果才算可用于辅助志愿填报？建议满足：

- 211+ 专业组的 80% 预测区间覆盖率不低于 70%—80%。
- 保底组失效率低于 5%。
- 热门工科组重大误判可解释。
- 新增/拆分/合并组不强行给高置信度。
- 模型相对去年位次延续基线有明显改进。

如果保底失效率超过 10%、热门组覆盖率低于 60%、大量特殊类型组被误混入普通组，则模型不可靠，只能作为资料整理工具。

## 八、风险分层规则

先定义预测最低位次区间：

```text
L = Q10
M = Q50
U = Q90
```

这里 `L` 是偏前移、竞争更强的边界，数值更小；`U` 是偏后移、相对宽松的边界，数值更大。考生位次为 `r_c`，数值越小越好。

录取概率：

```text
p = P(T_g >= r_c)
```

推荐规则如下。

### 冲

考生位次略差于预测区间宽松边界，即：

```text
U < r_c <= U(1+0.08)
```

或校准概率约 15%—35%。可填，但只能放前部，不可依赖。

### 稳中有险

考生位次落在预测区间内，即：

```text
L < r_c <= U
```

或概率约 35%—70%。如果组内热门专业占比高、计划减少、专业组拆分，应向“冲”降级。

### 稳

考生位次优于竞争边界，但优势不大，即：

```text
0.90L < r_c <= L
```

或概率约 70%—85%。可作为中部志愿，但仍要看调剂风险。

### 保

考生位次明显优于竞争边界，即：

```text
0.80L < r_c <= 0.90L
```

或概率约 85%—95%。要求专业可接受、计划稳定、无重大章程限制。

### 垫底保底

考生位次大幅优于竞争边界，即：

```text
r_c <= 0.80L
```

且概率大于 95%，专业和调剂均可接受，不能是明确禁忌专业组，不能是中外合作/专项/民族班/预科等特殊口径。

### 不建议填报

以下情况不建议填报：

- 概率低于 15%。
- 专业组含明确不能接受的专业且需要服从调剂。
- 学费、身体条件、单科成绩、外语语种不符合。
- 中外合作费用不可接受。
- 专项、民族班、预科不符合资格。
- 该组虽低分但专业后悔风险极高。

### 数据不足，不可判断

新增组、重度拆分组、合并组、无历史投档、无出档、计划极少、信息缺失、专业组口径无法对应。可人工评估，但模型不得给高置信度。

## 九、2026 年应用流程

### 阶段 1：数据收集

目标是拿到 2024、2025 历史数据和 2026 填报前数据。要做的是下载或录入一分一档、投档线、招生计划、专业组专业明细、章程、学校城市信息。产出 `raw_data` 文件夹。完成标准是每个文件有来源、日期、URL 或截图记录。常见错误是用第三方数据替代官方数据、把征集投档线混入第一次投档线。

### 阶段 2：数据清洗

目标是统一字段和口径。要做的是清洗院校代码、专业组代码、分数、位次、批次、科类、计划性质。产出 `clean_score_rank.csv`、`clean_admission_line.csv`、`clean_plan.csv`。完成标准是同一院校专业组能跨年匹配。常见错误是把历史类混入物理类，把专项计划混入普通批。

### 阶段 3：建立基础表

目标是形成可建模数据库。要做的是生成上面八张表。产出 database 或 Excel 工作簿。完成标准是每个 `group_id` 唯一，且能关联学校、城市、专业组、投档线和计划。常见错误是专业组代码相同但年份不同、院校不同，导致误关联。

### 阶段 4：建立基准模型

目标是得到可解释基线。要做的是实现去年位次延续、计划修正、专业热度修正、城市层级修正。产出 `baseline_model.py` 和 `baseline_prediction_2025.csv`。完成标准是能用 2024 预测 2025。常见错误是直接用分数差预测。

### 阶段 5：建立机器学习模型

目标是辅助捕捉非线性关系。要做的是训练岭回归、LightGBM/CatBoost、分位数回归、方向分类模型。产出 `ml_model.py`、`feature_importance.csv`、`prediction_interval.csv`。完成标准是模型输入不含泄漏特征。常见错误是样本太少却使用复杂模型并输出伪精确概率。

### 阶段 6：回测

目标是判断模型是否能辅助填报。要做的是用 2024 预测 2025，计算 MAE、MAPE、RMSE、覆盖率、保底失效率、冲刺命中率，并对 211+ 和热门工科单独评估。产出 `backtest_report.md` 或 xlsx。完成标准是误差、覆盖率、误判案例清楚。常见错误是只报告“准确率”而不报告保底失败。

### 阶段 7：人工修正

目标是处理模型不擅长的异常。要做的是人工检查章程、专业组拆分、合并、新增、学费、校区、禁忌专业。产出 `manual_adjustment.csv`。完成标准是每个修正都有理由。常见错误是人工随意改分，不留依据。

### 阶段 8：生成志愿候选池

目标是服务个人目标：广西物理类物化生，550—620 分，211 及以上优先。要做的是筛出 211+、双一流、强双非强势工科，重点保留计算机、电子信息、自动化、人工智能、软件工程、通信、数学、物理、工科类专业组。产出 `candidate_pool.csv`。完成标准是每个专业组都有风险档、专业接受度、调剂风险。常见错误是只按学校最低分选学校。

### 阶段 9：正式填报前更新 2026 数据

目标是用官方 2026 招生计划和一分一档替换预估。要做的是导入 2026 计划、考生真实位次、2026 一分一档，重新计算所有预测。产出 `prediction_2026_final.csv`。完成标准是所有候选组使用 2026 正式计划。常见错误是继续使用 2025 计划。

### 阶段 10：输出最终志愿表

目标是形成可填报的 40 个院校专业组序列。要做的是按冲、稳中有险、稳、保、垫底保底组合排序，并核查每个组内专业是否可接受、是否服从调剂。产出 `final_choice_table.xlsx`。完成标准是没有资格不符、专业禁忌误放、章程限制遗漏。常见错误是保底志愿也放自己不能接受的专业组。

## 十、最终交付物

最终应产出 12 类文件。

1. 数据源清单：`source_list.xlsx`，记录来源、网址、发布日期、获取日期、可信级别。
2. 原始数据文件夹：`raw_data/`，保存官方原始 PDF、Excel、网页截图或 HTML。
3. 清洗后的 CSV/Excel：`clean_data/`，包括一分一档、投档线、招生计划、专业组明细。
4. 字段说明文档：`data_dictionary.md`，解释每个字段含义、类型、口径。
5. 模型公式说明：`model_specification.md`，写清所有公式、变量、适用条件。
6. Python 建模脚本：`scripts/`，包括 `data_cleaning.py`、`feature_engineering.py`、`baseline_model.py`、`ml_model.py`、`calibration.py`、`backtest.py`。
7. 回测报告：`backtest_report.xlsx` 或 pdf，包含整体和分层指标。
8. 院校专业组风险表：`group_risk_table_2026.csv`，包含每个专业组的预测位次区间、概率、风险解释。
9. 个人志愿决策表：`personal_choice_table.xlsx`，基于考生位次、专业偏好和禁忌筛选。
10. 冲稳保组合建议：`strategy_summary.md`，说明冲多少、稳多少、保多少、垫底多少。
11. 不建议填报清单：`not_recommended.csv`，列出原因，如专业禁忌、资格不符、学费不可接受、章程限制。
12. 数据不足清单：`insufficient_data.csv`，列出新增、拆分、合并、无历史、计划极少的专业组。

## 十一、特别注意事项

第一，新高考专业组口径变化是最大风险。专业组不是学校，不能把学校最低分当专业组最低分。

第二，不能把旧高考理科数据直接和新高考物理类院校专业组数据混用。旧高考“院校+专业”口径与新高考“院校专业组+专业”口径不同。

第三，本科普通批、提前批、专项计划、中外合作、民族班、预科班、征集志愿不能混在一起建模。

第四，招生计划变化会显著影响位次。计划减少的热门工科组尤其容易前移。

第五，专业组拆分和合并会导致历史不可比。尤其是把计算机、电子信息单独拆出时，不能沿用原大组最低位次。

第六，热门专业组容易突然上涨。计算机、人工智能、电子信息、软件工程、通信、自动化在 211+ 学校中尤其要谨慎。

第七，冷门专业组可能有保底价值，但也可能有隐藏风险。比如土木、材料、化工、环境、生物、食品、海洋、农林、矿业、地质等，如果明确不接受，就不能拿来当保底。

第八，中外合作必须单独建模。它受学费、合作项目、校区、是否可转专业影响，不能和普通专业组比较。

第九，民族班、预科班、国家专项、地方专项、高校专项必须单独建模，并先检查资格。没有资格则直接不可填。

第十，第三方数据可能有转录错误。第三方可用于辅助抓取，但最终必须回到广西招生考试院、广西阳光志愿、广西高考指南、阳光高考和高校章程核验。

第十一，机器学习可能过拟合。若模型比去年位次延续模型好不了多少，应优先采用更简单、更可解释的模型。

第十二，预测不能替代人工判断。最终填报前必须逐个核查专业组内专业、调剂范围、章程限制、身体条件、单科成绩、外语语种、学费、校区。

## 十二、虚构示例：完整流程演示

以下是虚构示例，不对应任何真实高校数据。

假设考生为广西物理类物化生，2026 年高考 590 分。根据 2026 官方一分一档表，590 分同分区间为 17800—18200 名。为了保守，取考生位次：

```text
r_c = 18200
```

虚构院校专业组 A：某 211 大学 101 专业组，包含计算机类、电子信息类、自动化、数学类、物理学类。2024 年最低投档分换算位次为 21000 名，招生计划 40 人；2025 年最低投档分换算位次为 18500 名，招生计划 36 人；2026 年计划 42 人。专业组结构基本稳定，热门专业占比从 2025 年的 0.70 升至 2026 年的 0.75。学校为 211，城市热度较高。

第一步，输入数据：

```text
R_{2024}=21000, R_{2025}=18500
P_{2024}=40, P_{2025}=36, P_{2026}=42
H_{2025}=0.70, H_{2026}=0.75
```

第二步，去年位次延续模型：

```text
\hat R^{base}_{2026}=18500
```

第三步，两年趋势模型。2024 到 2025 从 21000 前移到 18500，竞争增强。假设回测后趋势收缩系数取 `\lambda=0.5`：

```text
\hat R^{trend}=18500+0.5(18500-21000)=17250
```

这表示如果趋势继续，可能前移到约 17250 名。

第四步，计划修正。2026 计划从 36 增至 42，计划增加 16.7%。假设历史回测估计 `\gamma=0.30`：

```text
\hat R^{plan}
=
17250 * exp(0.30 * log(42/36))
≈ 18050
```

计划增加使最低位次后移，风险略下降。

第五步，专业热度修正。热门专业占比从 0.70 升至 0.75，假设回测估计热度系数会带来约 3% 的前移：

```text
\hat R^{hot}=18050 * 0.97 ≈ 17508
```

第六步，机器学习模型输出。LightGBM 辅助模型预测中位位次为 17600；分位数模型输出：

```text
Q10=16000, Q50=17600, Q90=19500
```

第七步，预测区间解释。该专业组 2026 最低投档位次可能落在 16000—19500 之间，中位数约 17600。区间较宽，说明存在一定不确定性。

第八步，录取概率。考生保守位次为 18200。因为：

```text
16000 < 18200 < 19500
```

考生位次落在预测区间内部，但略差于中位数 17600。用校准模型估计：

```text
P(T >= 18200) ≈ 35%-50%
```

第九步，冲稳保判断。该组不能算“稳”，应判为“稳中有险偏冲”。如果非常喜欢该组，可放在志愿表前部；但不能作为保底。

第十步，给考生的建议。虚构建议如下：该 211 专业组专业匹配度高，包含计算机、电子信息、自动化等目标专业，值得填；但由于 2024—2025 已经明显前移，且热门专业占比继续提高，风险不低。建议将其放在“冲/稳中有险”区域，同时在后面配置若干预测区间明显覆盖自己位次、专业也能接受的 211 冷门工科或强双非强势工科组。不能用护理、农林、矿业、地质、土木、材料、化工、环境、生物、食品、海洋等明确排斥的组做保底；如果必须用冷门组保底，也要先确认能接受调剂结果。

整体结论是：这套模型的核心不是算一个“2026 分数线”，而是建立“院校专业组最低位次分布 + 考生位次相对位置 + 概率校准 + 人工规则核验”的决策系统。对广西物理类物化生、550—620 分、211 及以上优先、偏计算机电子信息工科的目标，最重要的是：用位次，不用分数；用专业组，不用学校最低分；特殊类型单独建模；保底必须专业可接受；模型必须先通过 2024 预测 2025 的回测，再用于 2026 正式数据更新。
